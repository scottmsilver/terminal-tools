Understanding the Scaling Chain

Your client screen (physical pixels)
  └─ CRD client (may scale to fit window)
       └─ Server framebuffer (?x? via xrandr)
            ├─ Xft.dpi → WezTerm reads this → font_size = 11 * (DPI/96)
            ├─ xsettingsd Xft/DPI → Chrome, GTK apps use this for rendering scale
            ├─ i3 config font → bar text, window titles (manual pt size)
            └─ i3 border/gaps → window chrome (manual px size)

Key insight: Xft.dpi and xsettingsd must agree, or WezTerm and Chrome will scale differently. In the old script's laptop mode, Xft.dpi=192 but xsettingsd=216 — that's a 12.5% mismatch!
